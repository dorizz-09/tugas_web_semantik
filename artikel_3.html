Definisi dan Cara Kerja Nama Domain

Apa itu nama domain, bagaimana cara kerjanya, dan pentingnya dalam mengidentifikasi sumber di internet, serta perbedaan antara nama domain dan URL.

Apa itu nama domain?

Nama domain adalah string teks unik yang digunakan untuk mengidentifikasi sumber tertentu di internet. Ini bisa berupa apa saja, mulai dari situs web hingga email atau server FTP.

Saat Anda mengakses sebuah situs web, nama domainnya diubah menjadi serangkaian angka (alamat IP) untuk berkomunikasi dengan server situs web guna mengirim permintaan dan menerima konten situs web.

Nama domain dirancang untuk manusia

Semuanya secara teoretis bisa ditangani hanya dengan alamat IP ketika Anda ingin membuka halaman web di browser. Faktanya, menambahkan nama domain menambah langkah ekstra karena perlu ada cara untuk mengonversi dari nama domain ke alamat IP (lebih lanjut tentang ini di bawah).

Jadi, mengapa kita menggunakan nama domain sama sekali?

Bayangkan jika, alih-alih mengingat nama setiap orang, Anda harus mengingat nomor telepon atau nomor jaminan sosial mereka setiap kali Anda ingin membicarakan mereka. Atau bayangkan masuk ke toko elektronik dan menemukan tidak ada merek, nama perangkat, atau nama model. Semuanya hanya memiliki nomor seri yang harus Anda ingat jika ingin membelinya.

Kita, manusia, jauh lebih baik dalam mengingat nama daripada mengingat rangkaian angka. Itulah mengapa kita berbicara tentang nama merek dan model yang menarik, dan itulah mengapa kita menggunakan nama domain daripada alamat IP.

Apa yang bukan nama domain

Sementara Anda dapat menggunakan nama domain untuk situs web atau akun email, itu bukanlah hal yang sama dengan situs web atau email itu sendiri. Dengan kata lain, semua situs web dan akun email memerlukan nama domain, tetapi memiliki nama domain tidak secara otomatis berarti Anda memiliki situs web atau akun email.

Nama domain dapat dibagi menjadi dua (atau kadang tiga) bagian, masing-masing dipisahkan oleh titik.

Domain tingkat atas

Domain tingkat atas (TLD) adalah bagian dua atau tiga huruf di sebelah kanan titik. Dalam nama domain "Codecademy.com," ".com" adalah TLD-nya. TLD umum lainnya termasuk:

  • .edu (untuk institusi pendidikan)
  • .org (biasanya untuk organisasi nirlaba)
  • .gov (untuk organisasi pemerintah)
  • .net
  • .io

TLD di atas dikenal sebagai TLD generik karena dapat digunakan oleh siapa saja di seluruh dunia. Tetapi ada juga TLD spesifik negara, seperti:

  • .ca (Kanada)
  • .mx (Meksiko)
  • .de (Jerman)
  • .cn (Cina)
  • .jp (Jepang)

Meskipun orang lebih bebas memilih bagian lain dari nama domain mereka (jika belum digunakan), TLD cenderung lebih ketat. Misalnya, Anda harus menjadi institusi pasca-sekolah yang terakreditasi untuk mendaftarkan domain dengan TLD .edu di AS. Dan, sebagian besar negara membatasi TLD mereka untuk warganya sendiri atau bisnis yang terdaftar di dalam perbatasan mereka.

Domain tingkat kedua

Domain tingkat kedua (2LD) adalah yang ada di sebelah kiri TLD. Dalam contoh "Codecademy.com," "Codecademy" adalah 2LD-nya.

Domain tingkat ketiga

Terkadang, sebuah nama domain memiliki tiga bagian. Yaitu, tiga nama yang dipisahkan oleh dua titik. Pikirkan tentang situs web komersial Inggris, yang selalu diakhiri dengan .co.uk. Jadi, untuk situs Google di Inggris, "Google.co.uk," ".uk" adalah TLD, ".co" adalah 2LD, dan "Google" adalah domain tingkat ketiga (3LD).

Sebuah 3LD juga dapat digunakan untuk menunjuk ke subdomain. Misalnya, jika Anda menggunakan Wikipedia dalam bahasa Inggris, Anda akan mengunjungi "en.Wikipedia.org." Di sini, ".org" adalah TLD, "Wikipedia" adalah 2LD, dan "en" (untuk Inggris) adalah 3LD. Subdomain sering digunakan untuk membedakan antara layanan yang berbeda. Misalnya, ambil subdomain Google seperti docs.google.com atau drive.google.com.

Apa perbedaan antara nama domain dan URL?

Anda mungkin melihat ke bilah alamat dan bertanya-tanya apakah itu semua bagian dari nama domain Codecademy. Apa yang sebenarnya Anda lihat adalah uniform resource locator atau URL.

URL berisi nama domain ("Codecademy.com") dan dua informasi penting lainnya: protokol transfer dan jalur. Perlu dicatat bahwa beberapa browser, seperti Google Chrome, tidak lagi menampilkan URL lengkap di bilah alamat. Jika Anda menggunakan Chrome dan tidak dapat melihat URL lengkapnya, klik kanan bilah alamat dan pilih "Selalu Tampilkan URL Lengkap."

Protokol transfer (yang "https://" di sebelah kiri nama domain) adalah konvensi yang digunakan antara komputer dan server di internet (dalam kasus sebagian besar lalu lintas web, protokol HTTPS yang sangat umum).

Nama jalur berisi informasi tentang halaman atau file tertentu di dalam situs web. Misalnya, semua yang Anda lihat di bilah alamat Anda di sebelah kanan "Codecademy.com/" adalah jalur URL.

Sama seperti jalur file di komputer Anda, jalur URL untuk artikel ini adalah unik dan memungkinkan Anda untuk dengan mudah mengakses halaman ini di kemudian hari atau membagikannya dengan teman-teman Anda di media sosial.
